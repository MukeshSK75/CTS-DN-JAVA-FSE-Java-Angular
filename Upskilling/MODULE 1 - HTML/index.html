Chrome DevTools Guide

Practical steps to inspect, debug, and experiment with this portal page.

1

Open DevTools

Press F12 (or Ctrl+Shift+I / Cmd+Option+I on Mac). The DevTools panel opens docked to the right or bottom of your browser.

2

Inspect Element — Live CSS Editing

Right-click the welcome banner above → Inspect. In the Styles pane, change background from #1a3c5e to any hex color and watch the page update live. Try #e05c2a.

3

Console Tab — View Script Logs

Open the Console tab. Click the button below to trigger a log. You'll see structured output from this page's JavaScript in real time.

4

Sources Tab — JavaScript Breakpoints

Go to Sources → Page → (this file). Find the submitRegistration() function. Click the line number to add a breakpoint. Then submit the registration form — DevTools pauses execution so you can inspect variables in the Scope panel.

5

Network Tab — Monitor Requests

Switch to Network, reload the page (Ctrl+R). See every resource loaded. Since this is a single self-contained file, only one document request appears — confirming zero external dependencies.

6

Application Tab — Check localStorage

Go to Application → Storage → Local Storage → file:// (or your localhost URL). After saving preferences (Task 8), you'll see the preferredEvent key and its value stored there. You can edit or delete keys directly from this panel.

7

Device Toolbar — Responsive Testing

Press Ctrl+Shift+M to toggle the device toolbar. Switch between phone presets (e.g., iPhone 14 Pro) to see how the portal's responsive grid and form layouts adapt to narrow screens.
